Minecraft Bedrock Custom JSON UI Syntax Reference

This document describes the structure, properties, and usage examples of the custom JSON UI system used for defining UI elements in Minecraft Bedrock Edition.

TOP-LEVEL STRUCTURE

Each UI file contains an object with key-value pairs:

{
  "element_id": { UIElementDefinition },
  ...
}

COMMON PROPERTIES

All UI elements may use the following:

  • type: string // 'panel', 'button', 'stack_panel'
  • size: [number, number] // width, height in pixels
  • offset: [number, number] // x, y offset from screen center
  • color: string // hex color (e.g. '#ff00ff')
  • alpha: number // 0 to 1, transparency
  • visible: boolean // true or false

TYPES OF ELEMENTS

1. PANEL

A simple colored box.

"my_panel": {
  "type": "panel",
  "size": [300, 100],
  "offset": [0, 0],
  "color": "#00ffcc",
  "alpha": 0.8
}

2. BUTTON

An interactive clickable button.

Additional properties:

  • text: string
  • controls: array of control states
  • bindings: array of bindings
"my_button": {
  "type": "button",
  "size": [120, 40],
  "offset": [100, 0],
  "text": "Click Me",
  "bindings": [
    { "binding_name": "my_panel", "binding_name_override": "#visible" }
  ],
  "controls": [
    { "default": { "color": "#888" } },
    { "default_hover": { "color": "#aaa" } },
    { "pressed": { "color": "#555" } },
    { "checked": { "color": "#4caf50" } }
  ]
}

3. STACK_PANEL

A layout container that arranges child controls in a row or column.

Additional properties:

  • orientation: 'horizontal' or 'vertical'
  • contain: boolean
  • controls: array of inline or referenced elements
"my_stack": {
  "type": "stack_panel",
  "offset": [0, 0],
  "size": [300, 300],
  "orientation": "horizontal",
  "contain": true,
  "alpha": 1,
  "controls": [
    { "button1": { "type": "button", "size": [20, 20], "text": "One" } },
    { "button2": { "type": "button", "size": [40, 20], "text": "Two" } }
  ]
}

BINDINGS

Used for dynamic visibility toggles.

"bindings": [
  {
    "binding_name": "target_id",
    "binding_name_override": "#visible"
  }
]

CONTROL STATES (FOR BUTTONS)

Each control is an object with a single key:

  • default, default_hover, pressed, pressed_hover, checked, checked_hover
"controls": [
  { "default": { "color": "#999" } },
  { "pressed": { "color": "#f00", "size": [50, 50], "offset": [0, 0] } }
]

These can also include additional nested UI definitions (e.g., popups).

NAMESPACING (EXTERNAL REFERENCES)

A stack_panel control may refer to an external file’s path:

"controls": [
  { "@external_namespace.element_name": null }
]

You must supply externalDefs when calling render(ui, externalDefs).

DEFAULT CENTERED ORIGIN

All offsets are interpreted from the CENTER of the screen.

Example:

offset: [50, 0] → 50px to the right of center.
