Minecraft Bedrock Custom JSON UI Syntax Reference

This document describes the structure, properties, and usage examples of the custom JSON UI system made by [me](https://www.planetminecraft.com/member/ashura_tepes/) inspired mostly by mcbe json ui.

TOP-LEVEL STRUCTURE

Each UI file contains an object with key-value pairs representing UI elements:

{
  "element_id": { UIElementDefinition },
  ...
}

Where each element is identified by a unique element_id and defined by its respective UIElementDefinition.

COMMON PROPERTIES

All UI elements may use the following properties:

  • type: string // 'panel', 'button', 'stack_panel', etc. Defines the type of UI element.
  • size: [number, number] // [width, height] in pixels. Defines the dimensions of the element.
  • offset: [number, number] // [x, y] offset from the center of the screen. Defines the position relative to the screen center.
  • color: string // Hex color (e.g. '#ff00ff'). Defines the color of the element.
  • alpha: number // 0 to 1, transparency. Defines the opacity of the element, where 0 is fully transparent and 1 is fully opaque.
  • visible: boolean // true or false. Controls the visibility of the element.

TYPES OF ELEMENTS

1. PANEL

A simple colored box that can be used for backgrounds or containers.

"my_panel": {
  "type": "panel",
  "size": [300, 100],
  "offset": [0, 0],
  "color": "#00ffcc",
  "alpha": 0.8
}

2. BUTTON

An interactive clickable button. Buttons can trigger actions like opening menus, toggling visibility, etc.

Additional properties:

  • text: string // The text displayed on the button.
  • controls: array of control states. Defines how the button will appear in different states (e.g., normal, hovered, pressed).
  • bindings: array of bindings. Specifies what elements or properties are affected when the button is interacted with.
"my_button": {
  "type": "button",
  "size": [120, 40],
  "offset": [100, 0],
  "text": "Click Me",
  "bindings": [
    { "binding_name": "my_panel", "binding_name_override": "#visible" }
  ],
  "controls": [
    { "default": { "color": "#888" } },
    { "default_hover": { "color": "#aaa" } },
    { "pressed": { "color": "#555" } },
    { "checked": { "color": "#4caf50" } }
  ]
}

3. STACK_PANEL

A layout container that arranges child controls in a row or column. Useful for organizing elements.

Additional properties:

  • orientation: 'horizontal' or 'vertical'. Defines the direction in which the child elements are laid out.
  • contain: boolean // true or false. Controls whether the stack panel will constrain its children to fit within its bounds.
  • controls: array of inline or referenced elements. Defines the child elements inside the stack panel.
"my_stack": {
  "type": "stack_panel",
  "offset": [0, 0],
  "size": [300, 300],
  "orientation": "horizontal",
  "contain": true,
  "alpha": 1,
  "controls": [
    { "button1": { "type": "button", "size": [20, 20], "text": "One" } },
    { "button2": { "type": "button", "size": [40, 20], "text": "Two" } }
  ]
}

BINDINGS

Used for dynamic visibility toggles and other UI behavior connections between elements.

Bindings can toggle properties like visibility, text, or color of elements based on user interactions or other conditions.

"bindings": [
  {
    "binding_name": "target_id",
    "binding_name_override": "#visible"
  }
]

The binding connects the target element’s property (in this case, visibility) to another UI element’s state.

CONTROL STATES (FOR BUTTONS)

Each button can have different visual states based on its interaction with the user (hover, pressed, checked, etc.).

  • default: The button's appearance when it is not being interacted with.
  • default_hover: The button's appearance when the mouse is hovering over it.
  • pressed: The button's appearance when it is being clicked or pressed.
  • pressed_hover: The button's appearance when it is pressed and the mouse is still hovering over it.
  • checked: The button’s appearance when it is in a checked or toggled state.
  • checked_hover: The button’s appearance when it is in a checked state and the mouse is hovering over it.
"controls": [
  { "default": { "color": "#999" } },
  { "pressed": { "color": "#f00", "size": [50, 50], "offset": [0, 0] } }
]

These can also include additional nested UI definitions (e.g., popups).

NAMESPACING (EXTERNAL REFERENCES)

UI elements can reference external definitions to load additional UI elements or properties from other JSON files.

"controls": [
  { "@external_namespace.element_name": null }
]

When using external namespaces, you must provide an externalDefs parameter when rendering the UI to include the external files.

DEFAULT CENTERED ORIGIN

All offsets are interpreted from the center of the screen.

Example:

offset: [50, 0] → 50px to the right of center.
